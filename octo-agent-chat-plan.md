  Inter-Agent Communication

  Inter-agent communication works through three mechanisms:

  1. Group Chat — When you type in a group chat, the message is sent to ALL member agents' PTY terminals via gateway send(). Each agent works independently but receives the same instruction.
  2. Brief All — The "Brief All" button in group headers triggers the supervisor's briefing engine. For each group member, it sends a brief request via the gateway with sources listing all other members. The supervisor uses Claude API to generate a summary of what the other agents have been doing (reading from their memory.md files) and injects it into the target agent's context.
  3. Conflict Detection — The conflictDetector.ts tracks file ownership per session. If two agents touch the same file, it emits a conflict event + hard stop to prevent corruption.
